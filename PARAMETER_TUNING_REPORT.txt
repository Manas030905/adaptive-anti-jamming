KEY FINDINGS
================================================================================

1. LMS ALGORITHM - STABLE BASELINE
   Performance: Moderate, positive SNR in favorable conditions
   Speed: Fastest (0.2-0.3 seconds)
   Best For: Real-time applications, embedded systems
   
   OPTIMAL PARAMETERS:
   - Step-size (mu): 0.01 - 0.015
   - Filter Order: 16 - 32
   - Works best with: Higher input SNR (>=15dB), clean reference signals


2. NLMS ALGORITHM - NORMALIZED, REQUIRES TUNING
   Performance: Variable, highly sensitive to step-size
   Speed: Moderate (0.3-0.4 seconds)
   Best For: Signals with varying power levels
   
   OPTIMAL PARAMETERS:
   - Step-size (mu): 0.3 - 0.5 (normalized, larger than LMS)
   - Filter Order: 24 - 32
   - Weight clipping: RECOMMENDED (+/- 5 to 10) to prevent divergence


3. RLS ALGORITHM - SUPERIOR PERFORMANCE ON NARROWBAND
   Performance: BEST on tone/narrowband interference (+1.79 dB SNR)
   Speed: Slowest (0.7-1.0 seconds)
   Best For: Narrowband interference, research applications
   
   OPTIMAL PARAMETERS:
   - Forgetting Factor (lambda): 0.85 - 0.99
     * 0.99 = Tracks slow changes (static)
     * 0.95 = Balanced tracking
     * 0.90 = Aggressive tracking
   - Filter Order: 16 - 24 (keep SMALL for stability)
   - P-matrix Regularization: 1e-5 to 1e-6 (prevents divergence)

================================================================================
PARAMETER TUNING GUIDE
================================================================================

STEP 1: Choose Filter Order
  Simple interference (single tone): 8-16
  Mixed interference: 24-32
  Complex broadband: 64+
  
  Rule: Smaller orders more stable, larger orders better adaptation

STEP 2: Select LMS Parameters
  For low SNR: mu = 0.005-0.01 (conservative)
  For high SNR: mu = 0.01-0.02 (faster)
  Trade-off: Smaller mu = slower convergence but more stable

STEP 3: Select NLMS Parameters
  Start with: mu = 0.3-0.5 (normalized, scale differs from LMS)
  Add weight clipping: Yes (+/- 5 to 10)
  If diverging: reduce mu or add more regularization

STEP 4: Select RLS Parameters
  For static interference: lambda = 0.99
  For dynamic interference: lambda = 0.90-0.95
  Critical: Keep filter order small (16-24 max)
  Add regularization: P = P + 1e-5*I each iteration


================================================================================
PROBLEMS FIXED
================================================================================

ISSUE 1: RLS/NLMS Divergence (Negative SNR)
CAUSE: Ill-conditioned P-matrix, exploding weights
SOLUTIONS IMPLEMENTED:
  - Added P-matrix regularization (1e-5)
  - Reduced filter order (from 64 to 16-24)
  - Added weight clipping for NLMS
  - Better lambda tuning
RESULT: RLS now shows +1.79 dB on tone jammer


ISSUE 2: Poor Reference Signal
CAUSE: Combined jammer not well-correlated with interference
SOLUTION: Test with individual jammer types (tone, noise, sweep)
RESULT: RLS performs excellently on narrowband, adequate on broadband


ISSUE 3: Filter Order Too Large
CAUSE: Large orders cause numerical instability in RLS
SOLUTION: Limited order to 16-32 optimal range
RESULT: Significant performance and stability improvement
